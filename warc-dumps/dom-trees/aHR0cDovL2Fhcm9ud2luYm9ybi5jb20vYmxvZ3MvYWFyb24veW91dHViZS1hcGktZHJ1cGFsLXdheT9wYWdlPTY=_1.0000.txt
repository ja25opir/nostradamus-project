		<p>I blogged briefly recently about <a href="http://aaronwinborn.com/blogs/aaron/youtube-expands-api-good-embedded-media-field">YouTube's new and improved API</a>. Soon thereafter, there was a <a href="http://groups.drupal.org/node/9829">flurry of discussion</a> about creating a Google Summer of Code project to integrate it with Drupal, including a <a href="http://groups.drupal.org/node/10295">proposal</a> that wasn't accepted, but I believe nothing further happened from that point.</p>
<p>Until now.</p>
<p><img src="http://aaronwinborn.com/sites/aaronwinborn.com/files/youtube-api-demo.png" align="right"></p>
<p>Brad Bowman (<a href="http://drupal.org/user/120651">beeradb</a>) has stepped up to the plate and created a YouTube API wrapper. He hasn't released it yet, though he indicated that he plans to early next week. I have had the honor of an early review, and it is nice to work with.</p>
<p>I set it up on a test server, and was able to upload a video to its file directory and have it also uploaded to YouTube, showing up in my user account almost immediately. The API also let me view my YouTube video user feed, so I could verify the results without leaving my site.</p>
<p>Additionally, when retrieving videos, you have access to everything stored at YouTube with that video, including video duration (as shown in the screen shot), description, and user comments.</p>
<p><strong>The next step,</strong> after Brad releases the module, will be to integrate it with the other modules that can benefit from it: <a href="http://drupal.org/project/emfield">Embedded Media Field</a>, <a href="http://drupal.org/project/media_mover">Media Mover,</a> and others. </p>
<p>The API itself is fairly easy to set up. You first need to get a Developer Key and Client ID from the <a href="http://code.google.com/apis/youtube/dashboard/">YouTube Developer's Dashboard</a>, and paste them into the administration screen for the module. Then you're off to go! It works out of the box from that point.</p>
<p>The module comes with an example page where you can test everything, although I expect that will be moved to an example module by the time of the release.</p>
<p>This is us moving one step closer to a semantic web, with interchangeable parts seamlessly woven together into a quilted garment that is no longer one-size-fits-all.</p>
