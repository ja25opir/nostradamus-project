  <h2><a href="/blogs/aaron/embedded-expression-engine-works" title="Embedded Expression Engine in the Works">Embedded Expression Engine in the Works</a></h2>

      <span class="submitted">Tue, 05/20/2008 - 15:35 — <a href="/users/aaron" title="View user profile.">aaron</a></span>
  
  <div class="content clear-block">
    <p><a href="http://aaronwinborn.com/node/22"><img src="http://aaronwinborn.com/sites/aaronwinborn.com/files/emexgine-step3-tn.png" align="right"></a></p>
<p>You want to support a provider not included automatically with <a href="http://drupal.org/project/emfield">Embedded Media Field</a>? Just can't wait for the maintainers of the module to write a new include file for FrenchBlingVideoMashup.TV? But you don't know RegEx parsing from red beets and parsnips?</p>
<p>Fret no more.</p>
<p>Soon, you will be able to build your own provider parser, with little more than the cut-and-paste you're already familiar with.</p>
<p>I am in the middle of creating Embedded Expression Engine (emexgine), which will compare the URLs and Embed codes from a third party content provider and turn them into a new option for your video, image, or audio fields.</p>
<p><a href="http://aaronwinborn.com/node/22">Read on</a> to find out more! (With cool, full-size, preview screenshots!)</p>

  <h2><a href="/blogs/aaron/rpg-achievements" title="RPG Achievements">RPG Achievements</a></h2>

      <span class="submitted">Thu, 05/01/2008 - 01:05 — <a href="/users/aaron" title="View user profile.">aaron</a></span>
  
  <div class="content clear-block">
    <p>So I have recently implemented and committed RPG Achieve in the <a href="http://drupal.org/project/rpg/">RPG</a> module series for Drupal. This allows games to record achievements of players, such as "You have slain 14 dragons", or "You were the first to summit Ploggers Peak". Next in that progression will be RPG Quest, which will allow achievements to be grouped together.</p>
<p>I've begun implementing achievements in <a href="http://spindowners.com/">Spindowners</a>, an upcoming sci-fi RPG being built with the RPG module. The game's shaping up nicely, and is currently in a closed Alpha. It's nearly ready for an open Alpha, with one character class at least.</p>
<p>Achievements may be defined in the administration pages, or through rulesets. Once defined, you only need to call rpg_achieve_set("achievement_1", $pc) to set the achievement. Each achievement has fine-tuned permissions, so that they may be visible to the player or not, and whether or not a message of achievement is sent, for instance. Obviously, each player character has their own achievements. Once Views 2 is out and RPG is ported to d6, I'll also hook it into the upcoming RPG Views.</p>
<p>Other new changes: I've ripped out the definitions from Rulesets into their own .inc files, for automatic inclusion on import, since those will only be called once. It's easy to limit character classes for user roles, etc., now, with hook_rpg_pc_types. I'm also in the middle of defining a few more useful core objects in Drudge, such as "Shuffled Event Lists" (which spout out occasional messages when appropriate, such as "A hawk cries in the distance," or "A meteor shoots across the night sky"). Along these lines, I've introduced rpg_trigger_all('event_name'), which will call rpg_trigger for all objects in scope.</p>
<p>Scope is another new thing, necessary for events to work properly. Now, when each event is called, the $rpg['scope'] array will hold all objects in scope for that event. By default, that will be the pc, the pc's containing room, and any objects invoked during an action. Other objects may be added or removed manually, but that should be sufficient in most cases.</p>
<p>Next on my plate: the holy grail: a single game page that never reloads, instead reloading just the blocks as needed. This allows for the creation of neato game UI's, battle music, and the ever-popular ad block from hell...</p>
<p><strong>IF YOU HAVEN'T ALREADY</strong>, make a character at <a href="http://drupalrpg.org/">Drupal RPG</a>. It's fun! Pointless right now, but you can see what all the fuss is about. Make sure to check out the snowbank for fun things to do...</p>

  <h2><a href="/blogs/aaron/jquery-media-release" title="jQuery Media Release">jQuery Media Release</a></h2>

      <span class="submitted">Sun, 04/06/2008 - 16:22 — <a href="/users/aaron" title="View user profile.">aaron</a></span>
  
  <div class="content clear-block">
    <p>I made an official release for the <a href="http://drupal.org/project/jquery_media">jQuery Media</a> module. This is a wrapper for the <a href="http://www.malsup.com/jquery/media/">jQuery Media</a> plugin. I have made this a full featured module, an aid for developers and themers. We are using it for the audio clips of <a href="http://airamerica">Air America Radio</a>.</p>
<p>From its help page:</p>
<p>The jQuery Media plugin will turn links to media into their embedded media equivalents, or into a modal overlay embedding the media when the link is clicked. Installing and configuring the module is only the first step: you must also manually theme the output of links in order to enjoy the benefits of this plugin.</p>
<p>Theming is outside of the scope of this help page. You might look at Drupal's Theme Developer's Guide for more information. However, once you are ready to theme your elements, you only need to add a class and invoke this plugin. For instance, if you have a File Field with a video, simply use something like the following: </p><div class="codeblock"><code><span style="color: #000000"><span style="color: #0000BB">&lt;?php<br> jq_add</span><span style="color: #007700">(</span><span style="color: #DD0000">'jquery_media'</span><span style="color: #007700">);<br>print </span><span style="color: #0000BB">l</span><span style="color: #007700">(</span><span style="color: #0000BB">$node</span><span style="color: #007700">-&gt;</span><span style="color: #0000BB">title</span><span style="color: #007700">, </span><span style="color: #0000BB">$node</span><span style="color: #007700">-&gt;</span><span style="color: #0000BB">field_file_field</span><span style="color: #007700">[</span><span style="color: #0000BB">0</span><span style="color: #007700">][</span><span style="color: #DD0000">'filepath'</span><span style="color: #007700">], array(</span><span style="color: #DD0000">'class' </span><span style="color: #007700">=&gt; </span><span style="color: #DD0000">'media'</span><span style="color: #007700">)); <br></span><span style="color: #0000BB">?&gt;</span></span></code></div><p></p>
<p>You may override the global defaults of this plugin when configuring. Additionally, you may override a global default on a case-by-case basis by including the override in an options array when invoking the plugin. For instance, you might override the background color of players on a specific page with </p><div class="codeblock"><code><span style="color: #000000"><span style="color: #0000BB">&lt;?php<br> jq_add</span><span style="color: #007700">(</span><span style="color: #DD0000">'jquery_media'</span><span style="color: #007700">, array(</span><span style="color: #DD0000">'bgcolor' </span><span style="color: #007700">=&gt; </span><span style="color: #DD0000">'#33FF00'</span><span style="color: #007700">)); <br></span><span style="color: #0000BB">?&gt;</span></span></code></div> <p></p>
<p>This will override the defaults of the plugin for every media link on that page, which may in turn override those individually (more on that later).</p>
<p>The full array of global overrides are as follows:</p>
<ul>
<li>'flvplayer' =&gt; The path to the Flash Video Player to be used, such as the JW FLV Player or Wimpy Rave.
</li><li>'mp3player' =&gt; The path to the MP3 Audio Player to be used, such as the JW FLV Player or Wimpy MP3 Player.
</li><li>'boxtitle' =&gt; When using the MediaBox option (the modal overlay), this will be the title displayed on that box after clicking the link.
</li><li>'loadingimage' =&gt; When using the MediaBox option, the path to this image will be displayed while loading the media after clicking the link.
</li><li>'media width' =&gt; The width the media will be displayed as.
</li><li>'media height' =&gt; The height the media will be displayed as.
</li><li>'mediabox width' =&gt; The width the MediaBox will be displayed as.
</li><li>'mediabox height' =&gt; The height the MediaBox will be displayed as.
</li><li>'autoplay' =&gt; If TRUE, then play the media as soon as loading it.
</li><li>'bgcolor' =&gt; This will be the background color for the media player, in hexidecimal code (#RRGGBB).
</li><li>'invoke media' =&gt; If TRUE, then cause all links with the 'media class' to automatically be turned into embedded media when invoking the plugin.
</li><li>'invoke mediabox' =&gt; If TRUE, then cause all links with the 'mediabox class' to automatically display a modal overlay box when clicked.
</li><li>'media class' =&gt; The class that will be turned into embedded media on invocation. It uses jQuery CSS notation, so should be something like 'a.media' for links.
</li><li>'mediabox class' =&gt; The class that will display a modal overlay box when clicked on invocation. It uses jQuery CSS notation, so should be something like 'a.mediabox' for links.</li></ul>
<p>Finally, you may override a global setting on an individual basis. The customizations here are extensive, and rather than repeating that here, you are directed to the <a href="http://www.malsup.com/jquery/media/">jQuery Media homepage</a> for more information. As an example, you might set the size of a specific file with the following, while leaving any other jquery media links on the page at their default. (Note that we change the class away from the default 'media' in this example. It also assumes we've already invoked it as above.): </p><div class="codeblock"><code><span style="color: #000000"><span style="color: #0000BB">&lt;?php<br> </span><span style="color: #007700">print </span><span style="color: #0000BB">l</span><span style="color: #007700">(</span><span style="color: #0000BB">$node</span><span style="color: #007700">-&gt;</span><span style="color: #0000BB">title</span><span style="color: #007700">, </span><span style="color: #0000BB">$node</span><span style="color: #007700">-&gt;</span><span style="color: #0000BB">field_audio_file</span><span style="color: #007700">, array(</span><span style="color: #DD0000">'class' </span><span style="color: #007700">=&gt; </span><span style="color: #DD0000">'media-audio'</span><span style="color: #007700">)); </span><span style="color: #0000BB">drupal_add_js</span><span style="color: #007700">(</span><span style="color: #DD0000">"$('a.media-audio').media( { width: 200, height: 15, autoplay: true } )"</span><span style="color: #007700">, </span><span style="color: #DD0000">'inline'</span><span style="color: #007700">); <br></span><span style="color: #0000BB">?&gt;</span></span></code></div><p></p>
<p>Note that by default, the module will add its default overrides inline. However, you may set it to output them to a dynamic javascript file, and may additionally wish to set the defaults to a static js file, once development is complete. You can alter those settings when configuring the module.</p>

  <h2><a href="/blogs/aaron/plans-jquery-media-and-jmedia-support" title="Plans for jQuery Media and jMedia Support">Plans for jQuery Media and jMedia Support</a></h2>

      <span class="submitted">Wed, 03/26/2008 - 02:15 — <a href="/users/aaron" title="View user profile.">aaron</a></span>
  
  <div class="content clear-block">
    <p>I've just created the <a href="http://drupal.org/project/jquery_media" target="_blank">jQuery Media</a> and <a href="http://drupal.org/project/jmedia" target="_blank">jMedia</a> wrapper modules for those two jQuery plugins. I am excited about both of them, which are similar. I'm already using jQuery Media for the audio player for <a href="http://airamerica.com" target="_blank">Air America Radio</a> (to be launched next week), and found out about jMedia today while researching other uses for the plugin.</p>
<p>Basically, these plugins will replace links or other text with Multimedia, automatically compatible with the browser. They both support audio and video, and come packed with cool features that I haven't even finished exploring yet.</p>
<p>I'll show you how it works next week after Air America's launch. But I'll let you know, it's really, really cool. Makes developing multimedia intensive sites super easy. And I'll definitely have to put it in <strong>Drupal Multimedia</strong>, my book that should be published this summer.</p>
<p>I plan to support one or both of them in <a href="http://drupal.org/project/emfield" target="_blank">Embedded Media Field</a> as well, as it should make theming that easier than it currently is. They both already support <a href="http://drupal.org/project/jq" target="_blank">jQ</a> out of the box, for easier administration.</p>
<p>And that's just the beginning...</p>

  <h2><a href="/blogs/aaron/drupal-will-explode-your-site-a-million-pieces-and-why-you-want-that" title="Drupal will Explode your Site into a Million Pieces, and Why You Want That.">Drupal will Explode your Site into a Million Pieces, and Why You Want That.</a></h2>

      <span class="submitted">Mon, 03/17/2008 - 16:53 — <a href="/users/aaron" title="View user profile.">aaron</a></span>
  
  <div class="content clear-block">
    <p><a href="http://groups.drupal.org/node/9633" target="_blank">Views as a Web Widget</a> has the potential to revolutionize the Internet, now that I think about it. Taking a hint from Steve Rubel of Micro Persuasion in <a href="http://www.micropersuasion.com/2008/03/the-future-is-w.html" target="_blank">The Future is Web Services, Not Web Sites</a>, we are entering a time where creating an API for embedding content within another site is becoming a standard way of sharing information.</p>
<blockquote><p>The leading players on the web all see the train coming. They are wisely creating APIs and turning themselves into plug-and-play services, not just big destinations. YouTube is just the latest to do so today. Amazon has S3. Google has OpenSocial and an extensive library of APIs. As does Microsoft. Facebook is allowing its applications to live outside the site. Twitter is an API first and (eventually) a business model second. Finally, the booming widget economy shows the promise of small content that can go anywhere.</p></blockquote>
<p>The <a href="http://groups.drupal.org/node/9633" target="_blank">Views as a Web Widget</a> project is being proposed for Google's 2008 <a href="http://code.google.com/soc/2008/" target="_blank">Summer of Code</a>. This project will allow an administrator to turn any view into a widget.</p>
<p>As I have experience with this, having implemented <a href="http://drupal.org/project/views_slideshow" target="_blank">Views Slideshow</a> (any view into a slideshow), <a href="http://drupal.org/project/emfield" target="_blank">Embedded Media Field</a> (embedding external "widgets" from YouTube, Flickr, etc.), and a <a href="http://maplight.org/widgets" target="_blank">widget for Maplight</a>, and am interested in being a mentor this year, I plan to offer to mentor that project, along with <a href="http://drupal.org/user/110386" target="_blank">Alex Urevick-Ackelsberg</a>.</p>
<p><a href="http://technology.timesonline.co.uk/tol/news/tech_and_web/article3532832.ece" target="_blank">Tim Berners-Lee spoke last week on the topic</a>, saying that the semantic web will supersede the monolithic giants of Facebook and Google. As Dries recently indicated at his <a href="http://www.archive.org/details/DrupalconBoston2008-TheStateOfDrupal" target="_blank">State of Drupal address</a> in Boston, the Drupal of the future will be semantic web.</p>
<p>The Drupal of the Future will explode your site into a million pieces. And if your site can handle that, it will thrive in that multifaceted environment.</p>

  <h2><a href="/blogs/aaron/youtube-expands-api-good-embedded-media-field" title="YouTube Expands API: Good for Embedded Media Field">YouTube Expands API: Good for Embedded Media Field</a></h2>

      <span class="submitted">Thu, 03/13/2008 - 02:02 — <a href="/users/aaron" title="View user profile.">aaron</a></span>
  
  <div class="content clear-block">
    <p>So YouTube just added some <a href="http://www.youtube.com/blog?entry=yFlR6EEySg8">new features</a> to their API:</p>
<ul>
<li>Upload videos and video responses to YouTube
</li><li>Add/Edit user and video metadata (titles, descriptions, ratings, comments, favorites, contacts, etc)
</li><li>Fetch localized standard feeds (most viewed, top rated, etc.) for 18 international locales
</li><li>Perform custom queries optimized for 18 international locales
</li><li>Customize player UI and control video playback (pause, play, stop, etc.) through software</li></ul>
<p>How does this apply to <a href="http://drupal.org/project/emfield">Embedded Media Field</a>? And how about to Drupal as a whole?</p>
<p>Embedded Media Field still needs to tie into the <a href="http://drupal.org/project/media_mover">Media Mover</a> module, firstly, to even allow uploading (harvesting) videos and sending them to YouTube (processing) before storing them in the content field. Once it does that, though, we can get a little smarter. Perhaps allowing users to post responses through the API, which would become YouTube videos showing up in response to a video there, and automatically also becoming a response to the Drupal content.</p>
<p>We can also tap into the custom queries when selecting video to embed.</p>
<p>Finally, we should be able to have more control over the UI, and could use jQuery to control the playback of the video, perhaps autoplaying a few seconds after the page load, or maybe jumping forward to our favorite part of the video. We'll have to see about that.</p>
<p>What does that mean for the Internet? As pointed out by Erick Schonfeld at <a href="http://www.techcrunch.com/2008/03/12/youtube-the-platform/">TechCrunch</a>, Google's just flexing their muscles a bit more, trying to ensure the bulk of video on the Internet goes through GooTube. At least Embedded Media Field is taking a more open approach, independently supporting many other providers, such as <a href="http://blip.tv">Blip.TV</a>, <a href="http://revver.com">Revver</a>, and <a href="http://brightcove.com">Brightcove</a>.</p>
<p>Aaron</p>
