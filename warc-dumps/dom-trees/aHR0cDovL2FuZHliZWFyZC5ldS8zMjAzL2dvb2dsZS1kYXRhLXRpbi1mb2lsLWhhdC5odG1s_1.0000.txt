    			<h1 class="entry-title">Google Data Tied Together – Get On Your Tin Foil Hat</h1>
<div class="entry-meta"><span class="meta-prep meta-prep-author">By </span><span class="author vcard"><a class="url fn n" href="http://andybeard.eu/author/Andy Beard/" title="View all posts by Andy Beard">Andy Beard</a></span><span class="meta-sep meta-sep-entry-date"> | </span><span class="meta-prep meta-prep-entry-date">Published: </span><span class="entry-date"><abbr class="published" title="2010-09-16T20:41:24+0000">September 16, 2010</abbr></span></div><!-- .entry-meta -->
				<div class="entry-content">

<p>A google engineer playing games with data is a bad thing.</p>
<p>It is bad when it happens with communications, especially when <a href="http://valleywag.gawker.com/5637234/gcreep-google-engineer-stalked-teens-spied-on-chats?skyline=true&amp;s=i">kids are involved</a>.</p>
<p>It is bad when it might be used for competitive reasons such as <a href="http://techcrunch.com/2009/04/07/shoemoney-sues-google-employee-for-adwords-violations/">Adwords data</a> that was <a href="http://www.shoemoney.com/2009/08/04/farrell-lawsuit-%E2%80%93-last-update/">eventually settled</a>.</p>
<h2>Get Your Tin Foil Hat On</h2>
<p>There are lots of reasons to use multiple Google accounts</p>
<ul>
<li>A perfect example would be to maintain an account for each domain you are likely to sell in the future as a master account for things like Google analytics. Then grant access to a user account for maintenance. This prevents one “spy vector” for competitors.</li>
<li>You can do similar for adwords if there is a need to share login data with a tool that doesn’t use an API with a central MCC account for clients.</li>
<li>You might want to run multiple branded Google buzz accounts</li>
<li>Personal vs Private – especially now Google is becoming more a social platform</li>
</ul>
<p>Google frequently claim that Analytics data isn’t used by the web spam team, and isn’t used for rankings, however for some, one small excerpt from the Gawker article should at least give them pause for thought.</p>
<p><img src="http://andybeard.eu/wp-content/uploads/google-accounts-tied-together.png" alt="google accounts tied together" title="google accounts tied together" width="564" height="201" class="aligncenter size-full wp-image-3204"></p>
<p>In some cases this is something you might expect – lots of people recently have been having their <a href="http://andybeard.eu/3100/google-adwords-rules.html">Adwords accounts banned</a> for even what seems innocent reasons – a poor interpretation of <a href="http://www.armandmorin.com/major-google-issue-affects/">bridge pages by Google adwords reps</a> (and terrible communication skills).<br>
Those people might want to get a new account, and Google will try their best to stop them.</p>
<h2>Spy Vector?</h2>
<p>I can just tell someone is going to ask about that. If you have 2 websites using Google analytics code from a single account, there are tools that can link those sites together.<br>
The same is true for IP addresses of hosting though that is less reliable, affiliate account details such as a Clickbank ID, and Google Adsense.</p>
<p>Google Adsense as far as I know still hasn’t any official supported way of making the code you place on one website anonymous to competitors.</p>
<h2>Of Course Google Know About All These Accounts</h2>
<p>SEOs have always assumed Google can tie different accounts together in some way – IP address, cookies etc</p>
<p>However I have never read an account of a Google engineer in any capacity actually doing it – almost as if the tools exist to do it for internal use. Data in a database that could potentially be tied together is one thing, tools to do it is quite another.
</p><div id="active-share-comment-marker"></div>
